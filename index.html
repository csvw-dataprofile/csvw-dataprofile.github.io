This specification defines the CSVW Data Profile: conventions and metadata patterns for describing tabular datasets using CSV on the Web (CSVW) and JSON-LD.

Status: Draft for review. Feedback via GitHub Issues.

Introduction

The CSVW Data Profile provides guidance for publishing consistent, machine-readable tabular data descriptions compatible with CSVW metadata and linked data practices.

This document is maintained by the csvw-dataprofile organization. It is published at https://csvw-dataprofile.github.io/.

Scope

  • Recommended CSVW metadata fields and patterns
  • JSON-LD contexts and terms for profiling
  • Validation expectations and examples

Examples

{
  "@context": "http://www.w3.org/ns/csvw",
  "url": "data.csv",
  "tableSchema": {
    "columns": [
      {"name": "time", "titles": "time", "datatype": "dateTime"},
      {"name": "height_cm", "titles": "height_cm", "datatype": "integer"}
    ],
    "primaryKey": "time"
  }
}

Conformance

Conformance requirements appear as RFC 2119/8174 keywords (MUST, SHOULD, MAY).
